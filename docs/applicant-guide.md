# Adaylar İçin Detaylı Başvuru Rehberi

Bu rehber, staj başvurusu yapmak isteyen öğrenciler için hazırlanmıştır.

## 📋 Başvuru Adımları

### 1. Issue Oluşturma

1. Repository'nin ana sayfasına gidin: `https://github.com/VB10/staj2026`
2. **Issues** sekmesine tıklayın
3. **New Issue** butonuna basın
4. **"Staj Başvuru Formu"** template'ini seçin

### 2. Başlık Formatı

Başlık **mutlaka** şu formatta olmalıdır:

```
AD-SOYAD [ALAN][STAJ YERİ][STAJ TİPİ][STAJ SÜRESİ]
```

#### Format Kuralları:

- **AD-SOYAD**: Tam adınızı yazın (örnek: `Ahmet Yılmaz`)
- **[ALAN]**: İlgilendiğiniz alan(lar) - virgülle ayırabilirsiniz
- **[STAJ YERİ]**: `uzaktan` veya `yüzyüze` (ikisini de seçebilirsiniz)
- **[STAJ TİPİ]**: `zorunlu` veya `gönüllü`
- **[STAJ SÜRESİ]**: `4-hafta`, `6-hafta` veya `8-hafta` (birden fazla seçebilirsiniz)

### 3. Geçerli Alanlar

Aşağıdaki alanlardan birini veya birden fazlasını seçebilirsiniz:

- `mobile` - Mobil uygulama geliştirme
- `backend` - Backend geliştirme
- `frontend` - Frontend geliştirme
- `pm` - Product Manager
- `qa` - Quality Assurance
- `game` - Oyun geliştirme
- `data-science` - Veri bilimi
- `data-analyst` - Veri analisti
- `database` - Veritabanı
- `embedded` - Gömülü sistemler
- `cyber-security` - Siber güvenlik
- `blockchain` - Blockchain
- `system` - Sistem yönetimi
- `networking` - Ağ yönetimi
- `hardware` - Donanım
- `sap-abap` - SAP ABAP

### 4. Örnek Başlıklar

✅ **Doğru Örnekler:**

```
Ahmet Yılmaz [backend,frontend][uzaktan][zorunlu][4-hafta]
Ayşe Demir [mobile][yüzyüze][gönüllü][6-hafta]
Mehmet Kaya [data-science,backend][uzaktan,yüzyüze][zorunlu][4-hafta,6-hafta]
Zeynep Öz [pm,qa][uzaktan][zorunlu][4-hafta]
```

❌ **Yanlış Örnekler:**

```
Ahmet Yılmaz [backend] [uzaktan] [zorunlu] [4 hafta]  // Boşluklar yanlış
Ahmet Yılmaz backend uzaktan zorunlu 4 hafta  // Köşeli parantez yok
Ahmet Yılmaz [Backend][Uzaktan][Zorunlu][4 Hafta]  // Büyük harf kullanılmış
```

### 5. Form İçeriği

Formu doldururken dikkat edilmesi gerekenler:

#### Kişisel Bilgiler

- **Ad Soyad**: Tam adınız
- **Okul**: Üniversite adı
- **Sınıf**: Sınıf bilginiz (örn: 3. Sınıf)
- **CV**: Google Drive linki veya PDF linki
- **Sigorta**: Okul tarafından karşılanıyor mu? (Evet/Hayır)

#### Kendin Hakkında

Bu bölümde:

- Kendinizi tanıtın
- İlgi alanlarınızı belirtin
- Hedeflerinizi paylaşın
- Motivasyonunuzu açıklayın

#### Teknik Bilgiler

- **İlgilendiğin Alanlar**: Bildiğiniz diller, frameworkler, teknolojiler
- **Örnek Projeler**: GitHub repo linkleri, demo linkleri, proje açıklamaları

### 6. Başvuru Sonrası

1. **Otomatik Kontrol**: Başlığınız otomatik olarak kontrol edilir
2. **Etiketleme**: Başvurunuz otomatik olarak etiketlenir
3. **Onay Mesajı**: Başarılı başvurularda onay mesajı alırsınız
4. **Eşleştirme**: Size uygun şirketler bulunduğunda bildirim alırsınız

## ⚠️ Hata Durumları

### Issue Hemen Kapanıyor

**Sebep**: Başlık formatı yanlış

**Çözüm**: 
- Başlığı kontrol edin
- Format kurallarına uygun olduğundan emin olun
- Örnekleri inceleyin
- Başlığı düzelttikten sonra issue otomatik açılacaktır

### Hata Mesajı Aldım

Hata mesajında belirtilen sorunları düzeltin:

- Eksik köşeli parantez
- Yanlış etiket isimleri
- Format hatası

### Format Kontrolü

Başlığınızın doğru olup olmadığını kontrol etmek için:

1. Tüm köşeli parantezlerin doğru yerleştirildiğinden emin olun
2. Etiket isimlerinin küçük harfle yazıldığını kontrol edin
3. Virgülle ayrılan değerlerin doğru formatta olduğunu kontrol edin

## 💡 İpuçları

### Başarılı Bir Başvuru İçin:

1. **Detaylı CV**: CV'nizde projelerinizi ve deneyimlerinizi belirtin
2. **GitHub Profili**: Aktif bir GitHub profiliniz olsun
3. **Projeler**: En az bir örnek projeniz olsun
4. **Açıklama**: Kendinizi iyi tanıtın, motivasyonunuzu belirtin

### Şirketlerle İletişim:

- Şirketler size issue üzerinden yorum yapabilir
- Size uygun şirketler bulunduğunda bildirim alırsınız
- Aktif olun ve yorumlara cevap verin

### Staj Bulduktan Sonra:

- Issue'yu kapatmayı unutmayın
- Başarı hikayenizi paylaşın (isteğe bağlı)

## 📊 İstatistikler
